print_header program

the print_header program reads informtion from the image header if the header exists and tries to extract useful information to help you understand your data. print_header can usually read headers from the following datatypes:
  • G.E. signa data (this is usually old data)
  • Siemens Somatom
  • G.E. genesis data (slicer was designed for this type of data originally but has been expanded to accomodate other formats.
  • DICOM data - the slicer program can usually read your dicom data directly. Please see
    Reading DICOM data into 3D Slicer But since dicom is a large and flexible spec the DICOM naming convention is confusing to print_header. if this is the case with your DICOM image names, you could try the make_dicom_links command
To run the program simply type (assuming that you have an image called I.001)

print_header I.001

There are several things to note about the output of print_header:
  • the information may not be correct, and will be inaccurate if you change the images without changing the headers.
  • not all of the parameters have meaning for all datatypes -
  • print_header can read your data in compressed (.gz or .Z) format as well as uncompressed

make_dicom_links

If print_header cant read your DICOM data, it is often because of the numbering schemes that some DICOM software uses. If this is the case with your data, try the following sequence of commands:
  • cd to the directory where your images are
  • make sure there are only your images and no other files!
  • type the command:
    make_spldicom_links
this will create a symbolic link for each of your DICOM images. If you now list the files in this directory, you will see something like the following:
lrwxrwxrwx 1 mark 28 Jul 5 19:11 J.097 -> long_dicom_image_name_97.dcm
lrwxrwxrwx 1 mark 28 Jul 5 19:11 J.096 -> long_dicom_image_name_96.dcm

These images numbered J.097,J.096, etc. point to your real DICOM data. This should allow the print_header program to work properly. If it still wont work, send email to Mark Anderson and he can help to sort out your data.
