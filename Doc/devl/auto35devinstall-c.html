This guide was written by Krishna Yeshwant and applies to Slicer version 2.

This is a brief step-by-step procedure for the Slicer2 installation starting from downloading VTK, Slicer2, and CMake. It may be missing steps, but ought to provide a general overview along with some specific CMake options. This procedure worked for me on a SunOS-5.8 machine in the SPL.

Preparation

  1. Download VTK 4.0 from Kitware.com
  2. Download Slicer2.0 from CVS
  3. Download CMake from Kitware.com
  4. Uncompress all.
  5. Install CMake

VTK Compilation

  1. Run ccmake in VTK directory and set the flags as follows.
    BUILD_EXAMPLES *ON
    BUILD_SHARED_LIBS *ON
    BUILD_TESTING *OFF
    CMAKE_CXX_FLAGS *
    CMAKE_C_FLAGS *
    CMAKE_INSTALL_PREFIX */usr/local
    DART_ROOT *NOTFOUND
    VTK_DATA_ROOT *NOTFOUND
    VTK_USE_ANSI_STDLIB *ON
    VTK_USE_HYBRID *ON
    VTK_USE_PARALLEL *ON
    VTK_USE_PATENTED *ON
    VTK_USE_RENDERING *ON
    VTK_USE_VOLUMEPRO *OFF
    VTK_WRAP_JAVA *OFF
    VTK_WRAP_PYTHON *ON
    VTK_WRAP_TCL *ON
  2. Configure and generate.
  3. Run make in VTK directory (takes a while)

Slicer 2 Compilation

  1. Go to slicer2/Base/builds
  2. mkdir solaris8 or redhat-7.3 (or whatever the target platform is)
  3. run ccmake ../..
    1. Ignore:
      Warning. VTK might be found on your system as different flavours: installed VTK or built VTK. Please make sure that the VTK_INSTALL_PATH or VTK_BINARY_PATH setting reflects which VTK you are planning to use, then set ONE of the USE_INSTALLED_VTK or USE_BUILT_VTK setting to ON.
    2. Change settings to:
      BUILD_TESTING ON
      CMAKE_CXX_FLAGS -O2
      CMAKE_C_FLAGS -O2
      CMAKE_INSTALL_PREFIX /usr/local
      DART_ROOT NOTFOUND
      USE_INSTALLED_VTK OFF
      VTK_BINARY_PATH /local/os/src/vtk-4.0
      VTK_INSTALL_PATH NOTFOUND
    3. Hit [c]onfigure once to bring up the following
      BUILD_SHARED_LIBS *ON
      USE_BUILT_VTK *ON
      WRAP_PYTHON *ON
      WRAP_TCL *ON
      BUILD_TESTING ON
      USE_INSTALLED_VTK OFF
      VTK_BINARY_PATH /local/os/src/vtk-4.0
      VTK_INSTALL_PATH NOTFOUND
    4. Hit [c]onfigure twice to bring up the following (again)
      BUILD_SHARED_LIBS *ON
      USE_BUILT_VTK *ON
      WRAP_PYTHON *ON
      WRAP_TCL *ON
      BUILD_TESTING ON
      USE_INSTALLED_VTK OFF
      VTK_BINARY_PATH /local/os/src/vtk-4.0
      VTK_INSTALL_PATH NOTFOUND
    5. Hit [c]onfigure thrice to bring up the following (yet again)
      BUILD_SHARED_LIBS ON
      BUILD_TESTING ON
      USE_BUILT_VTK ON
      USE_INSTALLED_VTK OFF
      VTK_BINARY_PATH /local/os/src/vtk-4.0
      VTK_INSTALL_PATH NOTFOUND
      WRAP_PYTHON ON
      WRAP_TCL ON
      • ALSO, [g] generate option appeared.
    6. Hit [g]enerate
    7. Type make in Base/builds/solaris8 directory (or whatever directory corresponds to the target platform)
