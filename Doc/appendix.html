Overview

The 3D Slicer has several initialization files. Each file is a set of key-value pairs where the key is the first word on the line, and the key's value is the remainder of the line. Key-value pairs are grouped into categories. Each category consists of a collection of lines inside parenthesis, as in the following example.

CategoryName (
key1 value1
key2 value2
)  

All colors are designated as 3 numbers between 0 and 1 for the red, green, and blue components.
Initialization File  File's Purpose 
slicer.init This file contains parameters that typically would be set once during program installation.
slicer.config This file contains parameters that control the program's appearance. Users may wish to change these parameters for giving a special demonstration, for example.
slicer.tabs This file configures the program's main menu and tabs. This file can be edited to reorder tabs, or add new ones.

Lut Category

The 3D Slicer uses various lookup tables (LUT) of colors for shading gray-scale data. Each table is specified here as a Lut category.

Key  Value 
idEach LUT has a unique ID number.
nameName for the LUT to appear on the user interface.
grayScale1 if the colors are all gray, else 0. If 1, then ignore hueRange, saturationRange, and valueRange
numberOfColorsSize of LUT
hueRangeLow and high values (between 0 and 1) that specify a color range from red (0) to blue (1)
saturationRangeLow and high values (between 0 and 1) that specify a saturation range from gray (0) to bright color (1)
valueRangeLow and high values (between 0 and 1) that specify a value range from dark (0) to intense (1)

Slice Category

There is a Slice Category for each of the 3 slices. The backVolID, foreVolID, and labelVolID specify which volumes are displayed on a slice. 0 specifies none. Numbers 1 or greater specify volumes in the order that they appear in the MRML file.

Key  Value 
idUnique ID number: 0, 1, or 2
visibility1 if visible, else 0
zoomZoom factor: 1, 2, 4, 8
driver"User" if the user controls the slice position, and "Locator" otherwise
orientOne of: Axial, Sagittal, Coronal, InPlane, InPlane0, InPlaneNeg90, Perp, Orig, Axi, Cor, Sag
offsetAxialOffset of axial slices in millimeters from the focal point. Similar parameters exist for each possible orientation.
backVolIDSpecifies the background volume on the slice with a number where 1 refers to the first volume in the input MRML file. 0 indicates that no volume should be used. Similar parameters exist for the foreground and label volumes.

slicer.tabs

This file configures the program's main menu and tabs. This file can be edited to reorder tabs, or add new ones.

One Tabs category exists with one key-value pair to specify which button on the main menu is initially pressed (designated as activeID. Then, there is one Tabs category for each button on the main menu. If there are more buttons then would fit, then some can exist in a menu of menu list of more options. These have their more key set to 1. The order of Tabs categories determines the order of buttons. Also, all the "more" Tabs should be listed last.

Tabs Category

Key  Value 
idUnique ID (typically same as the name).
nameName that appears on the button on the main menu
row1ListList of tabs that appear when the button is pressed
row1ShortSame as row1List but with shorter names to use when a tab is not pressed (only necessary when short of space)
row1,tabSelected tab on the first row of tabs.
row2ListList of tabs on the second row. If a second row is not needed, then leave this blank
row2ShortAbbreviated version of row2List
row2,tabActive tab on the second row of tabs
row"row1" or "row2", whichever row is visible
more0 for a button on the main menu, else 1 for an item in a list of more options on the main menu
